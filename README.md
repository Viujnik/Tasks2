# Tasks/Contracts Lab
## Запуск проекта
Для запуска, находясь в корне проекта, напишите команду ```uv run main```.

## Модели task'ов
В src/models.py вы можете увидеть простой dataclass Task(с id, типом задачи и payload - некоторые метаданные задачи). Далее идёт сам контракт для источников задач - TaskGiver, созданный с помощью Typing.Protocol. Он определяет метод get_tasks,который просто создает и складывает в список 5 задач соответствующего типа), и printf_task - форматированный вывод данных задачи. Всего 3 источника задач - FileSource, ConsoleSource и APISource.

## Симуляция
Под этим словом понимается вывод данных 5 задач каждого типа с помощью определенных ранее методов get_tasks и printf_task в функции give_engine.

## Тесты
Тесты разделены на 2 части в tests:
+ файл  test_models.py тестирует модели: работу get_tasks(функцией test_get_tasks) и форматированный вывод каждого типа задач.
+ файл test_engine.py мокает ввод и вывод и вызывает функцию give_engine, после чего проверяет, были ли вызвано 15 print'ов.

Для их запуска используйте команду ```pytest```, находясь в корне проекта. Можете прописать ```pytest --cov=src``` для проверки покрытия тестами.
